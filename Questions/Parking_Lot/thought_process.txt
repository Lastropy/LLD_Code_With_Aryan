Part 0: Core Idea
- Parking Lot to store multiple cars, across multiple floors.
- Each floor can have multiple slots, which can fit a car of certain size.
- So parking lot will have fixed capacity for each car type and total fixed capacity.
- When a car enters, it will take 1 slot.
- When a car leaves, it frees up 1 slot.
- When a car leaves, based on the time duration of parking and the type of car, it will be charged certain amount.
- Each car will belong to a owner, who will pay the parking fee on exit.

Part 1: State Functionalities
- Initialise a parking lot, having defined floors - each floor having certain vehicle type capacities.
- Method to check current capacity for the entering car type, and assign it a slot (if available).
- If vehicle is assigned slot, it will get a parking ticket.
- Managing parking space efficiently.
- Tracking the exit => collecting parking lot payment, then updating capacity of floor.

Part 2: Clarifying Questions
- How many vehicle types are there? (Slot types dependent on vehicle types)
- How many floors are there?
- How should we assign the vehicle to a slot, in case multiple slots are there?
- What if for a smaller vehicle, we only have bigger slots? Do we assign it that?
- What is the pricing strategy currently?
- What payment methods should we allow?
- What happens if a vehicle comes and there is no capacity?
- What are the core Functionalities?

=> We want a system where - 
- Supports multiple vehicle types (Car, Bike, Truck etc)
- Has different slot sizes, based on vehicle type.
- Allows multiple payment methods.
- Ensure smooth entry/exit with with accurate parking fee calculation.

=> So, to summarize, we want a Parking lot system where we have - 
- A parking lot with multiple slots
- Supports different vehicle types like car, bikes, trucks etc.
- Dynamic slot allocation based on vehicle size.
- Payment Processing with multiple payment modes (cash, card, upi etc)
- Entry tiket issuance, capacity checker, and exit validation.

=> Perfect, let's proceed.
